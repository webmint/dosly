**Description**:
This is the headline task. In one cohesive change, eliminate Flutter SDK
types from `lib/features/settings/domain/` and from
`lib/features/settings/data/`, switch theme persistence from `int` to
`String`, drop the `effectiveThemeMode`/`effectiveLocale` getters from
`AppSettings`, and confine all Flutter↔domain mapping to a single seam
in `lib/app.dart`.

Per MEMORY.md "Integration-gate task pattern" (Feature 007 Task 001 —
"Breaking the HomeBottomNav signature in Task 001 cascaded to home_screen.dart
and both test files in the SAME task, so no interim task saw broken state"),
this task is large by necessity: Dart's static type system will fail to
analyze unless ALL consumers of the changed shape are updated together.

This task does NOT update test files — those are deferred to Task 004
which acts as the integration gate. This task's `Done when` gates only
on `dart analyze` of `lib/`, not `flutter test`. **Tests will be
red-broken at the end of this task; that is intentional and Task 004's
problem.**

this task even though the files are open):
