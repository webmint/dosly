## Methodology

Adversarial mode — deliberate bias toward false positives over false negatives, but every finding is grounded in a verbatim quote from the actual code. Findings without grounding are discarded by Phase 4 validation. Confidence tiers indicate certainty. "Speculative" findings are hypotheses, not verdicts.

The "Failed verbatim-quote check" count is 1 (architect F13). One discard out of 43 candidate findings (~2.3%) is well below the >5 threshold that would suggest tone drift in agent prompts; the audit considers itself well-grounded.

Per Phase 4.3 rules, cross-agent consensus and recurring-issue tags are computed by exact-match hash keys, never by LLM "is this similar" judgment. This produces 0 auto-merges in this run despite obvious overlap, because agents pick different lines / phrasing for the same issue. The trade-off is intentional: it prevents confabulated consensus at the cost of report verbosity. The Top 10 priorities section above does narrative dedup at presentation time only — the Critical/High/Medium/Info buckets keep all agent views.
